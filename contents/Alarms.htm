Alarms Definition: Setting Up an Alarm Rule

Alarm Rules are what you use to determine when alarms are triggered in the system and by what, and when you are notified about them. You determine and insert the conditions that trigger the alarm; these conditions can be as simple or as complex as needed, including writing JavaScript code.

To create a new Alarm Rule, go to Alarms > Define Alarm Rule.

The page displays the configured alarm rules.

NOTE: This is a list of configured alarm rules and it does not display the triggered alarms. The triggered alarms are displayed on the Control Room page.

Click on + (New) to open the New Alarm Rule Editor.

Set applicable information in the Rule Definition tab:

Cluster: select the database source

Automation: the measurement is automatically collected from any source (DataColletor, forms, â€¦)

ETL Calculated: the measurement is calculated when importing the data and stored in the database

Runtime Calculated: the measurement is calculated within the mask and not saved on the database.

Unit:

Alarm class: select the type of alarm issue from the drop-down menu

Problem cause: choose what caused the problem

Company scope: choose a category for the alarm from the drop-down menu

Nature: select if itâ€™s a Weak Signal (such as a problem with a sensor or a non-critical alarm) or an Alarm (a critical issue that needs to be addressed)

Starting priority: set the priority level you want the alarm to have when it is triggered. (You can also set how the system escalates the priority level of the alarm if nothing is done to fix the issue that triggered the alarm: see Durations below.)
