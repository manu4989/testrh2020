6 Vessel Reporting

The HQ (Headquarters) Installation of the VRT (Vessel Reporting System) is a repository of all the reports filled in on the Ship Installation.

The Vessel Reporting functionality can be reached by clicking on the corresponding entry in the menu:

That includes the following sub-entries:

Company Reporting: here the user will be allowed to fill

Vessel Reporting reports that represent specific phases of the voyage, like Departure-Arrival-Noon

Ship Operation reports that represent specific operation performed during a voyage, like Bunker-Fuel Change

Log Books: in this section, the user will be allowed to report Deck & Engine Log Books (if needed)

Embarkation: this section is strictly connected to the Log Books one; considering the Company’s procedures, a specific person/role will be allowed to manage the embarkation/dis-embarkation of the crew members considering the port and date-time of the activities. The person that will be recognized as “embarked” by Performance, will be allowed to fill in the information in the reports

MRV/IMO-DCS: the user, ashore and onboard, will be able to download MRV and IMO-DCS reports in PDF format

All the mentioned reports will be filled onboard by the users that have the correct permissions set; all the reports will be visible also ashore.

6.1 Company reporting

With the Company Reporting module, you can consult all reports that are manually inserted on board the vessels during any voyage to check that data has been inserted correctly.

NOTE: The HQ Installation can see reports in read-only mode; changes cannot be made to reports from this Installation.

The Company Reporting menu entry consists of three sub-entries:

Voyage Summary: here you will be able to visualize all the reports of each specific voyage

Search: here you will be able to search for a specific type of reports during a period, or just simply search for all the reports that have been filled in in the period. It’s important to note that in Office you will also be able to search for any vessels of your fleet

Reports: the user will be allowed to download a specific report in PDF format (???)

6.1.3.2 Adding the First Report

The first report of the first voyage is extremely important because it will give to Performance the starting point of all the activities and checks related to voyage parameters like Fuel Consumption, R.O.B. (Remaining On Board), Sailing Time [all these parameters are automatically calculated and checked by the system during each voyage].

So, we strongly recommend inserting as a “first report of the first voyage” a PORT report; the Port report will photograph the situation and will let the user define the correct starting point of all the calculations.

To insert the first report, the user will need to perform the following steps:

Open the Voyage Summary

Click on New

Select Vessel Reporting

Select the Port report

The system will prompt the corresponding report to the user

The report has two main sections:

The higher section includes the header data

Date (UTC)

Elapsed time since last report (automatically calculated by the system)

Notes area

and can be defined as common for all the Voyage report.

The below section is split in different tabs and in each tab the user will be required to fill in different information (the information to be filled is defined considering Company’s needs and procedures)

The system will help the user compiling the report by highlighting the fields that have been defined as mandatory like the ones below

The system allows the user to save multiple times the report during the filling action, it is not required to fill in all the information at once. In this case, the report in the Voyage Summary will be highlighted with a specific icon and it will be saved by the system in DRAFT status

Until the misalignment is present, the system will show the triangle near the reporting date of the record.

When the user will open the report DRAFT, the system will highlight the field(s) that needs to be filled in or modified.

6.1.3.4 Adding a Ship Operation

The main difference between Vessel Reporting forms and Ship Operations is the fact that Ship Operations can be filled in at any time during any voyage, they don’t follow the sequence of the Vessel Reporting forms.

As said before, the Ship Operations will be configured and designed as per Company’s needs and procedures; there are some pre-defined components in the system that can be used in the definition, e.g. the R.O.B. component / the equipment fuel consumption component (directly linked to the equipment’s registry) and so on.

The steps to add a Ship Operation are similar to the ones expressed for Vessel Reporting reports:

Open Voyage Summary

Select the current voyage

Click New and select Ship Operations

Select the activity that needs to be compiled, e.g. Fuel Bunkering

The Ship Operation of Fuel Bunkering (the fuel’s list is directly linked to the fuel’s registry) will be opened

The user will need to fill in the correct Date (UTC) and add all the data related to the correct fuel. Click on “Save” button and the Ship Operation will be saved in the system.

NOTE: Since the Ship Operations are not included in the Voyage Report sequence, they can be inserted and deleted at any time of the voyage. Keep in mind that some Ship Operation are strictly connected to the data reported during the voyage and enhance them (e.g. the fuel characteristics), so deleting or adding a Ship Operation when the voyage is already closed could require some modifications to be done in the Voyage Reports too.

The look & feel of the report will be almost the same of the other reports, except for a button Acquisition from automation.

The field(s) that can be automatically filled in are highlighted with a different outline colour:

The user clicks on the button and the system will show a list of the samples (default sampling frequency of our Data Collector system is 5 minutes) considering a period of time between the date-time of the last compiled report and the date-time of the current report:

The first column represents the date-time of the sample; the other columns represent all the measures that are defined to be inherited from the automation.

The user can select the specific sample that needs to be user by pressing icon; the system will prompt a message asking for confirmation:

By pressing “Yes”, the user will confirm the sample selection and the values will be automatically updated in the form
